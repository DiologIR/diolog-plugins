## v2.0.0 — two detectors for the interaction-state and responsive blind spots

The v1.16 set closes static, resting-state drift. Two whole classes of "matches at the one
viewport I measured, but wrong" remained: what the UI does on **hover/focus** (a state the static
dump can never enter), and what it does at **other widths** (a single 1280 capture is blind to the
mobile layout and to whether the breakpoint TRANSITIONS even agree). v2.0.0 adds both to
`assets/diff/analyze.js`. Both capture in MODE A and compare in MODE B; both dedup a repeating root
cause to a few rows + a `[×N]` summary.

| # | What the differ was blind to | The mechanical check (where) | Real diolog example (live vs rebuild) |
|---|---|---|---|

## The standing principles these encode

1. **Reference the LIVE rendered site, never a re-served scrape** (#6). A runtime-hydrated framework resolves a different variant off-origin. This alone caused several "I matched it exactly but it's wrong".
2. **`getComputedStyle(element)` is blind to pseudo-elements and to non-CSS layers** (#9, #10). Fold pseudo borders/shadows; capture media-layer children.
3. **Computed style can match while the RENDER differs** — wrong glyph variant (cv11), variable-vs-static synthesis (#13), font-version width (#2), wrap balance (#5). Always carry **rendered geometry** (w/h/center-x), **glyph extent**, **line-count**, and **loaded faces** as ground truth, not just declared properties.
4. **Compare the SAME content on the SAME element type** (#7, #12). A node-boundary mismatch (dash-in-text, text-node-vs-button) makes a width/size comparison meaningless — pair to the styled box and slice to the same string.
5. **Gaps live between elements, not only inside boxes** (#11, #15). Check sibling gaps and spacing accumulation, not just each box's own padding.
6. **When the human eye and a measurement disagree, the eye wins — go LOOK / inspect the live element** (#9 was solved by the user reading `--border-color` in devtools). A confident wrong number is worse than no number; the fix is a better measurement, on the right node.
7. **Text-pairing alone is blind to NON-TEXT containers and to MISPAIRED repeated text** (#16, #19, #22, #17). A differ that only style-checks elements it matched BY TEXT never reaches a section/card/divider with no text of its own — so ALSO match containers STRUCTURALLY (path → geometry fallback) and compare their bg, full-bleed media layer, border (all sides, incl. `::after`), radius, shadow, and a page-wide hairline. And when the probe text is non-unique, DISAMBIGUATE by position before comparing, or every instance pairs to the first.
8. **`document.fonts.check` is necessary but NOT sufficient for "is the named face actually rendering"** (#20). A face can register as loaded yet not serve glyphs to rendered text (declared-vs-rendered drift). **The ground truth is a RENDERED-DOM metric, and it must be measured with a real DOM `<span>`, NOT canvas `measureText`** (v2.0.2). Canvas ignores `unicode-range` subsetting and does not replicate the browser's DOM font-matching cascade, so it cannot see a DOM-level fallback. Lay out a hidden span of a probe string twice — `'<Named>', <generic>` vs bare `<generic>` — for the same weight/style/size, against TWO distinct generics (monospace + serif); if the named width collapses to BOTH fallback baselines, the named family fell straight through to the fallback (`applies:false`). Await `document.fonts.ready` before probing. Compare the two SIDES' `applies` flags — flag only a disagreement, never a per-node absolute.
9. **Illustrations are excluded as content-noise, but their POSITION is still a defect class** (#21). Compare media/mockup-card geometry RELATIVE to a stable section anchor (an eyebrow/heading), never by absolute y — so page-wide vertical drift cancels and only a genuine in-section mispositioning fires. Position only, never internal content.
